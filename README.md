# gar-build-env

Gapless Agent Runtime 用の Codespaces/devcontainer ビルド環境です。

このリポジトリ自体はビルドツールをそろえる箱で、アプリやGAR toolsのソースは
`make setup` が `repos/` 配下へ clone します。`repos/` と `artifacts/` は作業領域なので
git 管理しません。

## Layout

```text
gar-build-env/
  .devcontainer/
  Makefile
  scripts/
  repos/                 # generated by make setup
    tools/
      gar-tools/
    apps/
      embedded-poc-app/
      gar-vibe-ui/
  artifacts/             # generated by make artifacts
```

## Setup

Codespaces 起動時は `.devcontainer/devcontainer.json` の `postCreateCommand` が
`scripts/post-create.sh` を実行します。これはOS packages、ESP-IDF、PlatformIOを準備し、
最後に `make setup` を呼びます。

手動で作業ツリーを用意する場合:

```bash
make setup
```

既存repoを明示的に最新化する場合:

```bash
make sync
```

`make setup` は存在しないrepoだけをcloneします。dirty worktreeを勝手にpullしないため、
作業中の変更を壊しにくい入口です。

## Linux Artifact Build

Linux / RasPi-compatible 向けのデモアプリとruntime shimをまとめます。

```bash
make artifacts
```

生成先:

```text
artifacts/from-codespace/
  artifact.json
  files/
    sensor_demo
    cuse_i2c
    cuse_spi
    web-bridge/
```

## M5StickC Firmware Build

Vibe Remote の M5StickC firmware をビルドしてartifact化します。

```bash
make m5stickc
```

生成先:

```text
repos/apps/gar-vibe-ui/vibe-remote/m5stickc-client/artifacts/
```

PlatformIO は Python 仮想環境 `~/.venvs/platformio` にインストールされ、
`~/.bashrc` に PATH が追加されます。
